Welcome to the Movie API Documentation

The MyFlix API allows users to access information about different movies, directors, and genres. Users are able to sign up, update their personal information, and create a list of their favorite movies.

Registered users can sign up, update their profiles, and maintain a personalized list of favorite movies.

GET /movies

Returns a list of all movies in the database.

Method: GET

Response:

[
  {
    "title": "Inception",
    "description": "A thief who steals corporate secrets...",
    "genre": { "name": "Sci-Fi", "description": "Science fiction" },
    "director": { "name": "Christopher Nolan", "bio": "...", "birthYear": 1970 },
    "releaseYear": 2010,
    "featured": true
  }
]

GET /movies/:title

Returns details about a single movie by title.

Method: GET

Response:

{
  "title": "Inception",
  "description": "...",
  "genre": { "name": "Sci-Fi", "description": "..." },
  "director": { "name": "Christopher Nolan", "bio": "...", "birthYear": 1970 },
  "releaseYear": 2010,
  "featured": true
}

GET /genres/:name

Returns genre details by name.

Method: GET

Response:

{
  "name": "Thriller",
  "description": "Suspenseful and intense"
}

GET /directors/:name

Returns director details by name.

Method: GET

Response:

{
  "name": "Christopher Nolan",
  "bio": "British-American filmmaker",
  "birthYear": 1970
}

POST /users

Registers a new user.

Method: POST

Request Body:

{
  "username": "johndoe",
  "password": "secure123",
  "email": "john@example.com",
  "birthday": "1990-01-01"
}

Response: Created user object

PUT /users/:username

Updates user information.

Method: PUT

Request Body:

{
  "email": "newemail@example.com",
  "password": "newpass456"
}

Response: Updated user object

POST /users/:username/movies/:movieID

Adds a movie to the user's list of favorites.

Method: POST

Response: Updated user object with favoriteMovies

DELETE /users/:username/movies/:movieID

Removes a movie from the user's list of favorites.

Method: DELETE

Response: Updated user object without the removed movie

DELETE /users/:username

Deregisters a user.

Method: DELETE

Response:

{
  "message": "User deleted",
  "user": { ... }
}
